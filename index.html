how to use

overview

A keyboard-driven field counter for traffic studies. Configure your intersection once in setup, then count by keystroke against time-lapse video. Three independent modes share the same session: vehicle (in/out by type), pedestrian (crossings by crosswalk and direction), and TMC (turning movement counts by approach leg).

basic workflow

  • Fill in the setup tabs — study parameters, intersection layout (crosswalks and approaches), vehicle types, and filenames.
  • Click start counting → to enter the counter.
  • Run your video. Use the video clock to track interval boundaries.
  • Press ↵ each time the clock crosses an interval boundary.
  • Count with the keyboard shortcuts shown in the reference bar. Switch between vehicle, pedestrian, and TMC modes with the toggle in the header — each keeps independent data.
  • Click ↓ in the header to export at any time.

navigation, editing & correction

▶advance to next interval ◀go back to previous interval Zundo last action Yredo

Edit a cell: click any count cell, type the corrected value, press Enter or Esc. Edited cells are highlighted.

Reset an interval: right-click any time label → reset. Single undoable action.

focus mode

Lock input to one target at a time — useful when traffic is heavy and you want to watch the video once per movement or crosswalk.

\toggle focus mode on/off [focus previous target ]focus next target (vehicle: also cycles type groups)

important

  • Counts live in this browser tab only — do not close or refresh during a session.
  • Window focus is required. If you click into a video player, keystrokes go there. A red banner appears when focus is lost — click anywhere on the counter to resume.
  • Export before ending any session.

study parameters

  • Start time — clock time of the first interval (e.g. 06:00). All row labels and CSV exports use real clock times.
  • Interval length — duration of each counting period. Choose a preset or enter a custom value in minutes.
  • Duration — total study length. Row count is calculated automatically as duration ÷ interval.

intersection builder

Choose a template (T / 4-way / 5-way), enter street names, then configure each leg independently. Each leg panel has two toggles:

  • Crosswalk — enables pedestrian counting on that leg. Set the crosswalk name, both direction labels, and key bindings.
  • Approach — enables turning movement counting from that leg. Check the destination legs you want to record.

Street names flow into destination labels (e.g. Thru (N) — Seminole Ave) and into default export filenames. The live diagram on the right shows crosswalk zones and approach arrows simultaneously as you build.

vehicle types

Configure 1“8+ types. Types are counted in groups of 4 — if you add more than 4, they split into groups and you cycle between them with [ / ]. Two FHWA presets are available. Keys must be unique within each group (different groups may reuse the same keys).

export filenames

Filenames default to Street1_Street2_TypeOfCount once street names are entered. All three can be changed mid-session via the ⚙ settings button.

key layout

Each vehicle type has an IN key (left hand) and an OUT key (right hand). Up to 4 types are shown at once; if you have more, they're split into groups — press [ or ] to cycle groups, or click the group buttons in the config bar.

default keys — group 1 (4 types)

in — left hand Atype 1 (IN) Stype 2 (IN) Dtype 3 (IN) Ftype 4 (IN) out — right hand Jtype 1 (OUT) Ktype 2 (OUT) Ltype 3 (OUT) ;type 4 (OUT)

Additional groups can reuse the same keys — pressing A in group 2 records group 2's type 1, not group 1's.

mid-session changes

Vehicle type labels can be edited from the config bar below the header without losing data. Key assignments can be changed mid-session via ⚙ settings — no restart required.

output

Two side-by-side tables show inbound and outbound counts. The CSV has separate INBOUND and OUTBOUND sections, one row per interval, one column per type, with totals.

structure

Counts crossing events by crosswalk and direction. Each crosswalk has two direction keys. All crosswalks share one unified table with a colored column pair per crosswalk.

intersection diagram

Open with the ⊞ button in the header. The floating diagram window shows your intersection with colored crosswalk bands, direction keys, and live counts for the current interval. Each keystroke briefly animates the matching count — peripheral confirmation without looking away from the video. Focus mode highlights the active crosswalk and dims the rest.

mid-session changes

Crosswalk names and direction labels can be edited from the config bar while counting. Key assignments can be changed via ⚙ settings. Template, slot assignments, and crosswalk count are fixed once counting begins.

output

CSV mirrors the on-screen table — time column, two direction columns per crosswalk, per-column and grand totals.

what is TMC mode?

Turning Movement Count mode records vehicles by approach leg and destination movement (left, through, right, U-turn). Configure approach legs in the intersection builder on the setup screen.

workflow

  • Switch to TMC in the mode toggle.
  • Select the active approach leg from the movement bar below the header, or use [ / ] to cycle between legs.
  • The keyboard reference shows one key per destination movement. Press a key to count a vehicle making that turn.
  • Vehicle type keys (IN keys from your vehicle setup) select which type is being counted for that movement.

turning diagram

The compact SVG diagram in the header shows all movements for the current approach with live counts. Open the full ⊞ popup for a larger view with a counts table by vehicle type and movement — useful on a second monitor positioned over the video.

destination labels

Destinations are labeled by movement and street name — e.g. Thru (N) — Seminole Ave, Left (W) — Morris Park Ave. Labels update live when you edit street names in setup.

output

TMC CSV exports one section per approach leg, with rows for each interval and columns for each destination → vehicle type combination.

exporting

Click the button in the header to download the current mode's data as a CSV. Vehicle and pedestrian modes export separately, using the filenames set in setup (editable mid-session via the ⚙ settings button).

Excel compatibility

Files include a UTF-8 byte-order mark so special characters like the en-dash in time ranges (e.g. 06:00 “ 06:15) display correctly in Excel without any manual import steps.

mid-session settings (⚙)

The settings button opens a small dialog for changing export filenames without interrupting your session. Structural changes — interval length, duration, type or crosswalk counts — require returning to setup, which warns before clearing counts.

data safety

  • All counts are held in memory in this tab. Closing or refreshing loses them.
  • Export regularly, and always before resetting or returning to setup.
  • For multi-session work, export and rename each CSV before starting the next.

settings

Label and filename changes apply immediately. To change interval length, duration, or pair counts return to setup — structural changes reset count data.
